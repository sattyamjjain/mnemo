# Fixed vs metric-driven consolidation — 2026-05-29

> Compares `ConsolidationPolicy::FixedSize` (v0.4.x default) against `ConsolidationPolicy::MaturityDriven` on a LoCoMo-style synthetic trace mixing mature (backdated, hit, edge-rich) and fresh (zero-access, no-edge) clusters. The maturity gate should consolidate the mature clusters and skip the fresh ones; the fixed gate consolidates everything.

## Setup

- Mature clusters per trial: 3.
- Fresh clusters per trial: 3.
- Facts per cluster: 4.
- Trials: 5 (medians reported).
- Mature-cluster backdate: 168h.
- Mature-cluster `access_count`: 12.
- Maturity arm: threshold = 0.50, balanced weights, default saturations.
- Fixed arm: `min_cluster_size = 3`, unconditional.
- Engine: in-memory DuckDB + USearch (dim=3, `NoopEmbedding` — vector + redundancy axes degenerate by design) + Tantivy BM25.

## Verdict

Pareto axes follow the user-specified framing of *recall retained* and *store-size reduction*. Selectivity is reported alongside as a supplementary axis (the design intent of the gate) but does not gate the Pareto verdict.

- **`recall_post (maturity) >= recall_post (fixed)`:** **yes**.
- **`active_bank_ratio (maturity) <= active_bank_ratio (fixed)`:** **no**.
- **Pareto win on (recall, store-reduction):** **no**.
- Supplementary — **`overreach (maturity) <= overreach (fixed)`:** **yes**.

## What this bench is NOT

- **Not a FluxMem reproduction.** [arXiv:2605.28773] is prior-art-only; mnemo's policy is a structural cousin (same four-axis intuition), not the FluxMem control loop.
- **`NoopEmbedding` makes the vector + redundancy axes degenerate.** Recall signal rides on BM25 + tag clustering; the needle string survives the consolidation bundle so BM25 still finds it. `recall_post` consequently tends to `1.0` in both arms. The headline numbers are `active_bank_ratio` and `overreach`.
- **Synthetic only.** A real-trace replay is a follow-up; this bench is a controlled comparison of the two gates on a repeatable trace.
- **Backdated `created_at` + explicit `access_count` drive a deterministic maturity outcome.** That is the bench's lever for reproducible mature / fresh classes; production traces arrive with whatever recency / hit-counts they arrive with.
- **`overreach` is measured by string-match on the fresh needle inside `[Consolidated from N memories] …` bundles.** Sufficient for this scenario, but not a general-purpose detector.
