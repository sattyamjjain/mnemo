## What v0.3.3 changes about benchmarking

The v0.3.0–0.3.2 runs published under `docs/benchmarks/` suffered from
two honest problems, both identified in the v0.3.1 benchmarks report
(`docs/benchmarks/2026-04-21-benchmarks-first-public.md`):

1. **Python `MnemoClient` did not attach a full-text index.** The
   binding constructed a `MnemoEngine` with `full_text = None`, so
   `strategy="hybrid_rrf"` silently collapsed to vector-only retrieval
   and every BM25-or-substring-shaped question missed. The fix is the
   `with_full_text: bool = True` kwarg (default on) in the PyMnemoClient
   constructor and the code now attaches a persistent Tantivy index at
   `<db-path>.tantivy`. See `python/src/lib.rs::MnemoClient::new`.
2. **Substring scoring undercounts LongMemEval.** LongMemEval's
   inferential questions have gold answers like *"Master of Business
   Administration"* when the supporting memory says *"MBA from Wharton"*.
   A verbatim-substring scorer records a miss; the retrieval actually
   worked. This is the "LLM-as-judge" line flagged as deferred in the
   v0.3.2 CHANGELOG. Task B lands:
   * `python/mnemo/benches/judge.py` — `LlmJudge` with `JudgeVerdict`
     (defaults `claude-haiku-4-5-20251001`, falls back to exact match
     when `ANTHROPIC_API_KEY` is missing).
   * `--judge=exact|llm` flag on the harness, default `exact` for
     backward-compatible CI.

## What is NOT in this document

Numeric recall@10 / MRR / latency for LoCoMo-MC10 and LongMemEval are
**deliberately deferred** to the first nightly run authenticated with
`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, and `HF_TOKEN` in the repo
secrets. The v0.3.3 code path is ready — running the numbers against
the public datasets requires:

* `pip install 'mnemo[benchmark]' anthropic datasets` (pulls `datasets`
  ~1 GB with its transitive PyArrow deps, not present on developer boxes
  by default);
* an HF account token for LongMemEval (rate-limited otherwise);
* an OpenAI key for `text-embedding-3-small` (Noop embeddings make
  strategy=hybrid_rrf collapse to BM25-only — Tantivy alone still beats
  the prior noop-only numbers, but the published comparison against
  Hindsight / Mem0 is with real embeddings);
* an Anthropic key for the judge (same setup Hindsight uses).

Both the workflow and the gate treat the first run as the baseline —
no regression error on day one. Subsequent runs compare recall@10
per-strategy and fail on a >3pp drop.

We are explicitly *not* publishing numbers from a partial local run
(no dataset, no judge key, Noop embeddings) and calling them
"Hindsight-parity". That would be the same over-claim the
`docs/benchmarks/2026-04-21-benchmarks-first-public.md` preface warned
about.

## How to run locally

```
# One-time setup
pip install 'mnemo[benchmark]' anthropic
cd python && maturin develop --release

# Small smoke run (no judge — cheap)
export OPENAI_API_KEY=...
python -m mnemo.benches.locomo_runner \
    --dataset longmemeval --max-examples 10 --judge exact

# Full run with LLM judge
export ANTHROPIC_API_KEY=...
export OPENAI_API_KEY=...
python -m mnemo.benches.locomo_runner \
    --dataset longmemeval --judge llm
```

Each run produces two artefacts under `docs/benchmarks/`:

* `YYYY-MM-DD-<dataset>.md` — human-readable table.
* `YYYY-MM-DD-<dataset>.json` — machine-readable sidecar for
  `check_bench_regression.py` + leaderboard charting.

## Regression gate semantics

Every nightly run invokes:

```
python .github/scripts/check_bench_regression.py \
    --dataset <name> \
    --baseline docs/benchmarks/baseline.json \
    --run-dir docs/benchmarks \
    --max-regression-pp 3.0
```

The script:

* Treats an empty baseline as the **first-run exception** (exits 0 and
  logs; the workflow is expected to commit the populated baseline on
  main in a follow-up step — tracked as a v0.3.4 polish).
* Fails with `::error` when any strategy drops recall@10 by more than
  3 percentage points vs the stored baseline.
* Treats a strategy that disappears from a run as a regression (it
  was dropped).
* Ignores strategies that appear in a run but not the baseline (new
  strategies are allowed to land without artificial gating).
