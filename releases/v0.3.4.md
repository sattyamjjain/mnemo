# Mnemo v0.3.4 — bench numbers + memory-tool 6-op server + R2 workspace

Released 2026-04-25. Tag: [`v0.3.4`](https://github.com/sattyamjjain/mnemo/releases/tag/v0.3.4). PR: [#45](https://github.com/sattyamjjain/mnemo/pull/45). Merge commit: `a9e48b5`.

## Highlights

- **Anthropic raw-API memory tool, full 6-op server.** Drop-in handler for Anthropic's `memory_20250818` surface — every Claude memory call lands as a Mnemo `MemoryRecord` with hash-chain + ACL coverage. Path-traversal hardened; spec-pinned return strings. `pip install 'mnemo[anthropic-memory-tool]'`.
- **Cloudflare R2 workspace backend.** R2-flavoured subclass of `S3Workspace`; same signed manifest + Ed25519 verification contract as the AWS path. `RemoteSnapshotSpec.backend == "r2"` for transparent dispatch through `MnemoSnapshotStore`. `pip install 'mnemo[openai-sandbox-r2]'`.
- **Public benchmark page laid out.** `docs/benchmarks/2026-04-25-mnemo-v0.3.4.md` carries Letta-parity reference rows (Hindsight 91.4% / 89.61%, Letta-Filesystem 74.0%, full-context 72.9% floor). Mnemo rows populate from the first authenticated nightly run — tracking [#44](https://github.com/sattyamjjain/mnemo/issues/44).

## Summary of changes

- **+32** Python tests for the memory-tool server (every op, every documented error, fixture round-trip, path-traversal rejection).
- **+5** Python tests for the R2 workspace (moto round-trip + S3-spec rejection + live-R2 opt-in).
- New extras: `mnemo[anthropic-memory-tool]`, `mnemo[openai-sandbox-r2]`.
- Issue [#39](https://github.com/sattyamjjain/mnemo/issues/39) rescoped to GCS + Azure Blob only after R2 landed.
- No breaking changes to v0.3.3 public surfaces.
